Width, Height e Size

Você precisa colocar as classes nas tags

w- e h- para definir largura e altura. size- para definir largura e altura iguais.

class="bg-gray-300">Normal
class="bg-gray-300 w-20 h-24">w-20 h-24
class="bg-gray-300 size-32">size-32

Unidades

Em geral, as unidades são geradas de 4px em 4px, então w-2 = 8px e w-5 = 20px. O Tailwind traduz o px para rem.
Existem exceções. w-1/2 = 50%, w-full = 100% e w-px é 1px.
existe alguns elementos como o border radius que eles não seguem esse padrão, eles usam o px mesmo

class="bg-gray-300 w-24"
class="bg-gray-300 w-1/2"

Padding e Margin

p- para padding e m- para margin.
py- para top/bottom, px- para left/right.
pt- = top, pb- = bottom, pr- = right, pl- = left.
É possível ter valores negativos para margin, como -mt-10 e auto mx-auto, não da para usar negativo no padding.

class="grid justify-items-start"
class="bg-gray-300 p-8"
class="bg-gray-400 mt-8 px-4 py-8"
class="bg-gray-200 pb-12 -mt-4 "
class="bg-gray-600 m-auto"

Border

Os valores de border são especificados em px, então border-4 é 4px de borda.
Os valores de bottom, top e etc são separados por -, por exemplo, border-b, e não borderb.

class="bg-gray-300 p-4 mb-4 border border-gray-800"
class="bg-gray-300 p-4 mb-4 border-b-4 border-gray-800"
class="bg-gray-300 p-4 border-x-4 border-gray-800"

border border-gray-800
border-b-4 border-gray-800

box-sizing

Diferente do HTML padrão, o Tailwind define os elementos com box-sizing: border-box;. Sendo assim, os valores de border e padding são absorvidos pelo valor do height e não somados ao mesmo. Podemos mudar para o comportamento padrão com box-content

class="bg-gray-300 border-gray-800 mb-4 p-4 border-y-4 h-16 box-border
class="bg-gray-300 border-gray-800 mb-4 p-4 border-y-4 h-16 pt-8 box-content"

p-4 border-y-4 h-16 box-border
p-4 border-y-4 h-16 pt-8 box-content
